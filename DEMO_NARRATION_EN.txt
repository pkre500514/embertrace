After a fire, crews should not have to rebuild every minute of an incident from memory. But the details they need are often scattered across helmet cameras, radio traffic, scene photos, dispatch records, and follow-up notes.

EmberTrace is an evidence-grounded fire incident report drafting prototype. It starts with a simulated, de-identified structure-fire case. The central rule is simple: evidence in, human judgment out. The system can organize a draft, but it must never invent facts, determine origin or cause, or submit a report automatically.

Here, four captured sources are available: a helmet-camera frame, a time-aligned radio transcript, scene photos, and a CAD dispatch record. The user can also add a redacted crew clarification. When I run the workflow, EmberTrace cross-checks the selected sources and creates a reviewable, NERIS-ready draft.

Every supported statement is linked back to the evidence that supports it. The incident summary cites dispatch and radio records. Operations cite the helmet camera and radio traffic. Observations cite the scene photos. When something is not supported, it remains visibly marked for review instead of being silently filled in.

The new AI-adapter path is intentionally testable without money or an API key. In the default no-key mode, EmberTrace runs a clearly labelled local simulation. It demonstrates the same selected-evidence, structured-draft, citation, and human-review contract, but it does not claim to be a GPT response.

When an agency configures an OpenAI key on its own server, that same adapter sends only the selected simulated evidence and explicitly entered redacted clarification to GPT-5.6 Terra. The browser never receives the key. Model output is validated, source-linked, and still cannot bypass the review gates.

Next, a reviewer opens the final report view. EmberTrace creates a local SHA-256 evidence receipt for the selected material, then requires two human confirmations. The application can prepare the report for supervisor review or print it to PDF, but it does not submit anything.

I built EmberTrace with Codex and GPT-5.6. Codex accelerated the workflow design, evidence and citation interactions, the optional adapter, safety boundaries, tests, and submission package. The key decision was to optimize for defensibility rather than fluent text generation.

EmberTrace is not a replacement for agency policy or human expertise. It demonstrates a safer AI-assisted documentation pattern: preserve the evidence, show uncertainty, and give crews time back.
